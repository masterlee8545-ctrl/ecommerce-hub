## 2. 어떤 문제를 푸는가 (What)

### 2.1 현재의 고통 (Current Pain)

**도구 분산 문제:**

| 단계 | 현재 사용 도구 | 문제 |
|---|---|---|
| 리서치 | 네이버 데이터랩 + 블랙키위 + 셀러몬 | 3개 이상 도구를 오감, 복사/붙여넣기 |
| 소싱 | 1688 + Excel 견적서 + 카카오톡 | 견적과 발주가 분리, 추적 안 됨 |
| 등록 | 쿠팡 윙 + 네이버 스마트스토어 + Excel | 등록 진행률 파악 불가 |
| 마케팅 | 쿠팡 광고 콘솔 + Excel 키워드 추적 | ROAS 하락 늦게 발견 |
| 작업 | 카카오톡 + 종이 메모 + Notion | 통합 안 됨, 누락 잦음 |
| 정산 | 각 플랫폼 정산서 + Excel 합산 | 수작업, 오류 잦음 |

**결과:**
- ⚠ **누락**: 발주 → 등록 단계 이행 시 디자인/촬영 작업이 잊혀짐
- ⚠ **지연**: 광고 ROAS 하락을 며칠 지나서야 발견
- ⚠ **신뢰성 저하**: Excel에 손으로 옮기다가 오타/누락
- ⚠ **의사결정 지연**: 한 상품의 전체 상태를 보려면 5개 도구를 열어야 함
- ⚠ **확장 불가**: 한 사람이 동시에 30개 상품을 추적하는 것이 한계

### 2.2 우리가 해결하는 것 (Solution)

**핵심 1: 6단계 파이프라인 통합**

```
① 리서치 → ② 소싱 → ③ 수입중 → ④ 등록 → ⑤ 판매중 → ⑥ 브랜딩
```

상품 한 건이 6단계 어디에 있든 한 화면에서 전체 상태 파악.

**핵심 2: 자동 작업 생성 (State Transition Hook)**

상태 전이 시 사람이 해야 할 작업이 자동 생성된다:

| 상태 전이 | 자동 생성 작업 | 담당자 |
|---|---|---|
| ② → ③ (소싱 → 수입중) | 통관/배송 추적 작업 | 박지연 |
| ③ → ④ (수입중 → 등록) | 상세페이지 디자인 + 상품 촬영 + SEO 키워드 + 광고 캠페인 (4종) | 박지연/김민수/이재홍 |
| ④ → ⑤ (등록 → 판매중) | 키워드 순위 모니터링 + 광고 예산 검토 | 이재홍 |
| ⑤ ROAS < 임계 | 광고 키워드 입찰 조정 작업 | 이재홍 |

**핵심 3: 신뢰도 추적 (Confidence Marking)**

모든 외부 데이터/추정값은 신뢰도가 명시된다:

| 마크 | 의미 |
|---|---|
| ✅ | API/공식 출처에서 확인됨 |
| 🟡 | Claude 또는 휴리스틱 추정값 |
| ❓ | 수집 실패 또는 미수집 |

**규칙:**
- 추정값(예: 마진율)은 최종 확정 전까지 명시적 경고
- 빈 데이터로 추정값을 은폐하는 것은 **즉시 불합격**

## 3. 왜 기존 솔루션으로는 안 되는가 (Why Build)

| 솔루션 | 한계 |
|---|---|
| Notion | 자동화 약함, API 통합 불가, 한국 이커머스 특화 부족 |
| 셀러몬 | 리서치 전용, 광고/작업 관리 없음 |
| 셀러쉐어 | 등록 자동화 전용, 리서치/광고 분리 |
| Excel | 자동화 0, 협업 어려움, 신뢰도 추적 불가 |
| Asana/Jira | 일반 작업관리 도구, 이커머스 도메인 지식 없음 |

**우리만의 차별점:**
1. 한국 이커머스 6단계 + 다중 회사 + 사람 작업 자동 생성을 한 시스템에
2. BW Rank API + 쿠팡 리뷰 API 직접 통합 (자체 인프라 활용)
3. 헌법 기반 안전 코드 (CLAUDE.md, ADR, 작업/검증 분리)
4. 모든 데이터에 신뢰도 추적 (할루시네이션 방지)

## 4. 성공 기준 (Success Criteria)

### 4.1 기능 측면

- [ ] 한 사용자가 50개 SKU의 6단계 상태를 5분 안에 파악 가능
- [ ] 상태 전이 시 작업 누락 0건 (자동 생성)
- [ ] 모든 외부 데이터에 신뢰도 표시
- [ ] 3개 회사 데이터 완전 분리 (RLS 검증)
- [ ] 쿠팡 1페이지 리뷰 분석 < 30초

### 4.2 데이터 측면

- [ ] 더미 데이터 0건
- [ ] Claude 추정값은 100% 🟡 표시
- [ ] API 실패 시 명시적 에러 (빈 배열로 은폐 금지)
- [ ] 새 키워드 분석 시 BW Rank API 호출 로그 남김

### 4.3 운영 측면

- [ ] `npm run gc` (lint + typecheck + test + build) 통과
- [ ] 모든 PR은 evaluator 검증 통과 (80점 이상)
- [ ] mistakes.md에 안티 패턴 등록 및 차단
- [ ] 회사별 RLS 정책 자동 테스트 통과

## 5. 명시적 비목표 (Non-goals)

이 시스템은 다음을 **하지 않는다**:

| 비목표 | 이유 |
|---|---|
| 쿠팡 윙에 자동 등록 | Akamai WAF로 차단됨 (수동 워크플로 유지) |
| 1688 직접 결제/주문 | 담당자 수동 처리, 시스템은 추적만 |
| 회계/세무 | 별도 솔루션 사용 |
| 3PL/물류 시스템 | 추후 연동 (Phase 4+) |
| 모바일 앱 | 웹 PWA로 충분 |
| 글로벌 마켓플레이스 | 한국 시장 집중 |

## 6. 단계 (Phases)

### Phase 1 — MVP (4주)
- 기본 6단계 파이프라인 CRUD
- 멀티 테넌트 (3개 회사 + RLS)
- 키워드 분석 (BW Rank API + 쿠팡 리뷰 API)
- 마진 계산기

### Phase 2 — 자동화 (8주)
- 작업 자동 생성 + 칸반 보드
- 광고 캠페인 관리 + ROAS 알림
- SEO 순위 추적 (BW Rank 폴링)

### Phase 3 — 통합 (12주)
- 정산 통합
- 브랜딩 단계
- 통계/리포트 대시보드

### Phase 4 — 확장 (TBD)
- 3PL 연동
- 추가 마켓플레이스
- 모바일 PWA 강화

## 7. 위험 (Risks)

| 위험 | 심각도 | 대응 |
|---|---|---|
| BW Rank API 다운 | High | 수동 입력 폴백 (manual fallback flag), 캐싱 |
| 쿠팡 리뷰 API 차단 | High | 캐싱 + 지수 백오프 재시도, 6시간 캐시 |
| 1688 단가 변동 | Medium | 마진 재계산 작업 자동 트리거 |
| 환율 변동 | Medium | 일일 환율 갱신 + 마진 재계산 |
| 데이터 손실 | Critical | Supabase PITR + 일일 백업 |
| 회사 간 데이터 누출 | Critical | RLS 정책 + 자동 테스트 |
| 작업 자동 생성 무한 루프 | High | 멱등성 키 (idempotency key) + 최대 재시도 |
