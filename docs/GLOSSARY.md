# GLOSSARY — 용어집 (사용자 친화 풀이)

> 이 문서는 CLAUDE.md §1 P-9 (전문용어 무설명 사용 금지)와 §11 (사용자 커뮤니케이션 규칙)의 운영 인프라다.
> 사용자(이재홍 대표, 비개발자)가 만나는 모든 기술 용어를 일상 한국어 비유로 풀어둔다.
> Claude가 사용자에게 보고할 때, 여기 등록된 단어가 처음 등장하면 한 줄 풀이를 함께 적어야 한다.
> 사용자가 모르는 단어를 물으면 즉시 여기에 추가한다.

| 항목 | 값 |
|---|---|
| 문서 버전 | 1.0 |
| 작성일 | 2026-04-07 |
| 관련 헌법 | CLAUDE.md §1 P-9, §11 |

## 0. 사용 규칙

- **알파벳/한글 가나다** 순서로 정렬
- 각 항목 형식:
  ```
  ### 단어 (영문)
  - **한 줄 풀이**: 일상 비유 한 줄
  - **정확한 정의**: 기술적으로 정확한 한 문장
  - **언제 등장**: 우리 프로젝트에서 어떤 작업할 때 나오는지
  - **사용자가 알아둘 것**: 의사결정에 영향이 있으면 적기
  ```
- **사용자가 물어본 단어는 무조건 추가** (P-9 §11.6)
- 같은 개념에 한국어/영어 둘 다 있으면 둘 다 등재 + 상호 참조

---

## 1. ㄱ

### 검색 인덱스 (search index)
- **한 줄 풀이**: 책의 맨 뒤 색인 — "○○ 단어가 몇 페이지에 있는지" 미리 만들어둔 표
- **정확한 정의**: DB가 특정 컬럼을 빠르게 찾기 위해 미리 만들어두는 보조 자료구조
- **언제 등장**: "이 컬럼에 인덱스 걸겠습니다" — 검색 속도 개선할 때
- **사용자가 알아둘 것**: 인덱스가 많을수록 검색은 빠르지만 데이터 추가/수정은 느려진다 (트레이드오프)

### 깃 (git)
- **한 줄 풀이**: 코드의 "이력서 + 타임머신". 누가 언제 무엇을 바꿨는지 다 저장하고, 과거로 돌아갈 수 있게 해주는 도구
- **정확한 정의**: 분산 버전 관리 시스템
- **언제 등장**: "커밋했어요", "푸시했어요", "브랜치 만들게요"
- **사용자가 알아둘 것**: GitHub은 git을 인터넷에 올려두는 서비스 (≠ git 자체)

---

## 2. ㄴ

### Next.js (넥스트JS)
- **한 줄 풀이**: 우리가 쓰는 웹사이트 만드는 틀. 페이지를 칸 나누어 담는 도시락통
- **정확한 정의**: React 기반 풀스택 웹 프레임워크
- **언제 등장**: 거의 모든 페이지/API 작업
- **사용자가 알아둘 것**: 우리는 14 또는 15 버전 사용 예정. 회사 데이터 확인 페이지(대시보드)를 만드는 데 가장 표준적인 도구

### npm
- **한 줄 풀이**: 외부 도구를 받아 우리 프로젝트에 설치하는 앱스토어 같은 프로그램
- **정확한 정의**: Node.js Package Manager
- **언제 등장**: `npm install`, `npm run dev`, `npm run gc`
- **사용자가 알아둘 것**: `npm install`은 "재료 사오기", `npm run dev`는 "테스트로 띄워보기"

---

## 3. ㄷ

### 데이터베이스 (DB, database)
- **한 줄 풀이**: 데이터를 표 형태로 안전하게 저장해두는 거대한 엑셀 파일집 (같은 시간에 100명이 봐도 안 깨지는)
- **정확한 정의**: 구조화된 데이터를 저장·검색·관리하는 시스템
- **언제 등장**: 거의 항상
- **사용자가 알아둘 것**: 우리는 PostgreSQL이라는 종류를 Supabase에서 빌려 쓴다

### Drizzle (드리즐)
- **한 줄 풀이**: 우리가 쓰는 ORM(코드↔DB 통역기) 이름. 가볍고 빠르고 TypeScript와 죽이 잘 맞음
- **정확한 정의**: TypeScript 친화적 ORM 라이브러리
- **언제 등장**: DB 작업 거의 모두
- **사용자가 알아둘 것**: ORM 항목 참고

---

## 4. ㄹ

### 라이브러리 (library)
- **한 줄 풀이**: 다른 사람이 만들어둔 코드 묶음. 우리가 가져다 쓰면 시간이 절약됨 (= 도서관에서 책 빌리기)
- **정확한 정의**: 재사용 가능한 코드 패키지
- **언제 등장**: "이 라이브러리 쓸게요", "package.json에 추가"

### 리졸버 (resolver)
- **한 줄 풀이**: 요청을 받으면 어떤 데이터를 돌려줄지 결정하는 함수 (= 식당 주방장에게 주문이 가는 것)
- **언제 등장**: API 만들 때

---

## 5. ㅁ

### 마이그레이션 (migration)
- **한 줄 풀이**: DB 구조를 안전하게 바꾸는 작업 + 그 변경 기록 (= 집 리모델링 + 도면 수정 이력)
- **정확한 정의**: DB 스키마 변경을 버전 관리하는 SQL 파일
- **언제 등장**: 새 표(테이블) 추가, 기존 표에 칸(컬럼) 추가
- **사용자가 알아둘 것**: 마이그레이션은 항상 새 파일로 추가한다 (옛 파일 수정 금지 — CLAUDE.md P-6). 잘못되면 과거 마이그레이션을 되돌려 복구 가능

### 멀티테넌트 (multi-tenant)
- **한 줄 풀이**: 한 시스템에서 여러 회사 데이터를 분리해서 보관 (= 같은 빌딩에 회사 3개가 입주, 서로 사무실 못 들어감)
- **정확한 정의**: 단일 인스턴스로 다수의 고객(테넌트) 데이터를 격리해 제공하는 아키텍처
- **언제 등장**: 회사 3개(BUYWISE 외 2개) 운영 — 거의 모든 작업
- **사용자가 알아둘 것**: 회사 간 데이터 노출은 우리 헌법 P-5의 최고 위반. 모든 쿼리에 회사 ID 강제

### 미들웨어 (middleware)
- **한 줄 풀이**: 요청이 들어오면 본 처리 전에 한 번 거르는 단계 (= 공항 보안검색대)
- **언제 등장**: 로그인 확인, 회사 ID 확인, 권한 체크

---

## 6. ㅂ

### 빌드 (build)
- **한 줄 풀이**: 우리가 쓴 코드를 사용자 브라우저가 이해할 수 있는 형태로 변환·포장하는 작업 (= 원고 → 인쇄본)
- **언제 등장**: 배포 전, `npm run build`
- **사용자가 알아둘 것**: 빌드가 통과해야 사용자에게 보낼 수 있다 (CLAUDE.md §2 Q2)

---

## 7. ㅅ

### 서버 (server)
- **한 줄 풀이**: 항상 켜져 있는 컴퓨터. 사용자가 접속하면 페이지를 만들어 보내줌 (= 24시간 식당)
- **언제 등장**: 거의 항상

### 스키마 (schema)
- **한 줄 풀이**: DB의 설계도 — 어떤 표에 어떤 칸이 있는지 정의 (= 엑셀 시트의 헤더 행 + 칸별 데이터 종류)
- **정확한 정의**: 데이터 구조 명세 (테이블, 컬럼, 타입, 제약 조건의 정의)
- **언제 등장**: "이 스키마로 가겠습니다", "스키마 변경 필요"
- **사용자가 알아둘 것**: 형제 프로젝트 InfoHub의 스키마는 우리가 함부로 가정하면 안 된다 (P-8). 항상 InfoHub가 알려준 그대로 따라야 함

### Supabase (수파베이스)
- **한 줄 풀이**: 우리가 쓰는 DB 호스팅 서비스 (= 인터넷에 있는 DB 클라우드)
- **정확한 정의**: PostgreSQL + 인증 + 스토리지 + Realtime 등을 묶어 제공하는 BaaS (Backend as a Service)
- **언제 등장**: DB 접속, 사용자 로그인 처리
- **사용자가 알아둘 것**: 매달 무료 사용량 한도 있음. 한도 넘으면 유료

---

## 8. ㅇ

### API (Application Programming Interface, 에이피아이)
- **한 줄 풀이**: 다른 서버와 약속된 방식으로 대화하는 창구 (= 전화 받는 창구)
- **정확한 정의**: 소프트웨어가 다른 소프트웨어와 통신하는 인터페이스
- **언제 등장**: 외부 데이터 가져올 때 (BW Rank, Coupang Review, InfoHub …)
- **사용자가 알아둘 것**: API 키는 비밀번호다. 노출되면 우리 회사 비용이 늘어남 (P-7)

### 인덱스 (index)
- **한 줄 풀이**: 검색 인덱스 항목 참고

### import / export
- **한 줄 풀이**: import = "이 책 빌려갑니다" / export = "이 책 빌려드립니다"
- **언제 등장**: 모든 코드 파일 첫 줄 근처
- **사용자가 알아둘 것**: 코드 간 연결 표현. 잘못 쓰면 "찾을 수 없음" 에러

### InfoHub (정보취합-2)
- **한 줄 풀이**: 우리 형제 프로젝트. 28개 사이트(유튜브, 네이버 블로그/뉴스, 구글 검색 등)에서 정보를 모아 AI로 분석해주는 도서관 같은 곳
- **정확한 정의**: BUYWISE 그룹의 정보 수집·분석 인프라 시스템
- **언제 등장**: Research(시장 조사) 단계, 소싱 단계
- **사용자가 알아둘 것**: 우리는 InfoHub의 "독자"다. 글을 쓰지 않고 읽기만 한다 (P-8, ADR-011). InfoHub가 가져온 정보는 모두 🟡 estimated(추정) 마크가 강제됨

---

## 9. ㅈ

### zod (조드)
- **한 줄 풀이**: 데이터 모양이 약속대로 왔는지 검사하는 도구 (= 택배 박스 검수원)
- **정확한 정의**: TypeScript-first 스키마 검증 라이브러리
- **언제 등장**: 외부 API 응답 받자마자
- **사용자가 알아둘 것**: zod 검증을 빼먹으면 InfoHub가 응답 형식을 바꿨을 때 우리 페이지가 통째로 깨진다 (B-001 사례)

### JWT (JSON Web Token, 제이더블유티)
- **한 줄 풀이**: 로그인했다는 증명서 토큰 (= 놀이공원 손목띠)
- **정확한 정의**: JSON 형식의 자기 검증 가능한 토큰 표준 (RFC 7519)
- **언제 등장**: 로그인 후 모든 요청
- **사용자가 알아둘 것**: 시간 지나면 만료. 자동 갱신 가능

---

## 10. ㅊ

### 캐시 (cache)
- **한 줄 풀이**: 자주 쓰는 정보를 빨리 꺼낼 수 있는 임시 보관함 (= 책상 위 자주 쓰는 펜)
- **정확한 정의**: 반복 조회되는 데이터를 메모리/스토리지에 임시 저장
- **언제 등장**: InfoHub 응답 캐싱 (6시간 TTL), API 응답 캐싱
- **사용자가 알아둘 것**: 캐시는 시간이 지나면 "오래된 정보"로 간주 (TTL). 우리는 6시간 후 ❓ 미확인으로 다운그레이드

### 컬럼 (column)
- **한 줄 풀이**: 표의 세로 칸 (= 엑셀 열). 예: "상품명", "가격" 같은 한 칸
- **언제 등장**: DB 표 설계할 때

### 쿼리 (query)
- **한 줄 풀이**: DB에 "이런 데이터 뽑아줘"라고 묻는 한 줄짜리 질문 (= 도서관 사서에게 "○○ 책 어디 있어요?" 묻는 것)
- **정확한 정의**: DB에 보내는 SQL 명령문
- **언제 등장**: 데이터 조회/저장/수정/삭제 시
- **사용자가 알아둘 것**: 우리 모든 쿼리는 "회사 ID 일치" 조건이 자동으로 붙도록 설계됨 (P-5 / RLS)

---

## 11. ㅌ

### 테이블 (table)
- **한 줄 풀이**: DB 안의 표 한 장 (= 엑셀 시트 1개)
- **언제 등장**: DB 작업 모두
- **사용자가 알아둘 것**: 우리 프로젝트는 21개 표를 만들 예정 (DATA_MODEL.md 기준)

### 토큰 (token)
- **한 줄 풀이**: 인증/식별을 위한 짧은 문자열 증명서 (= 손목띠, 입장권)
- **언제 등장**: 로그인, API 호출

### 트랜잭션 (transaction)
- **한 줄 풀이**: "이 작업들은 다 성공하든가, 다 실패하든가" 묶음 (= 은행 송금 — 출금만 되고 입금 안 되면 둘 다 취소)
- **언제 등장**: 여러 표를 동시에 수정해야 할 때
- **사용자가 알아둘 것**: 중간에 에러 나면 모두 자동 되돌림 → 데이터 깨짐 방지

---

## 12. ㅍ

### package.json (패키지 제이슨)
- **한 줄 풀이**: 우리 프로젝트가 쓰는 외부 도구 목록 (= 요리 재료 장보기 리스트)
- **언제 등장**: 거의 항상
- **사용자가 알아둘 것**: 새 라이브러리 추가하면 여기에 자동 기록됨

### 페치 (fetch)
- **한 줄 풀이**: 외부에서 데이터를 가져오는 동작 (= 심부름)
- **언제 등장**: API 호출

---

## 13. ㅎ

### 헌법 (Constitution)
- **한 줄 풀이**: `CLAUDE.md` 파일. Claude(AI)가 절대 어겨선 안 되는 최상위 규칙
- **언제 등장**: 거의 모든 작업의 출발점
- **사용자가 알아둘 것**: 이 헌법은 사용자(이재홍 대표)가 직접 손볼 수 있다. 헌법이 바뀌면 모든 작업 방식이 바뀐다

---

## 14. A-Z (영문 약어 모음)

### ADR (Architecture Decision Record)
- **한 줄 풀이**: "왜 이렇게 만들기로 했는지" 결정 기록서 (= 회의록)
- **언제 등장**: 큰 기술 결정 내릴 때
- **사용자가 알아둘 것**: ADR-001 ~ ADR-011이 있고, 우리 팀이 합의한 결정. 위반 시 코드 작성 금지 (P-4)

### ESLint (이에스린트)
- **한 줄 풀이**: 코드 작성 규칙을 자동으로 검사해주는 도구 (= 맞춤법 검사기)
- **언제 등장**: `npm run lint`

### MCP (Model Context Protocol)
- **한 줄 풀이**: Claude가 외부 도구를 부르는 표준 방식 (= 리모컨 + 적외선 표준)
- **언제 등장**: InfoHub와 대화할 때
- **사용자가 알아둘 것**: Anthropic이 만든 표준. InfoHub MCP를 통해 6가지 도구를 부를 수 있음

### ORM (Object-Relational Mapping)
- **한 줄 풀이**: 코드와 DB를 이어주는 통역기 (= 한↔영 자동번역기)
- **언제 등장**: 모든 DB 작업
- **사용자가 알아둘 것**: 우리는 Drizzle 사용. SQL을 직접 쓰지 않고 TypeScript 코드로 DB와 대화

### RLS (Row-Level Security)
- **한 줄 풀이**: 회사별로 자기 데이터만 보이게 하는 DB 차단막 (= 아파트 호수별 잠금장치)
- **정확한 정의**: PostgreSQL의 행 단위 접근 제어 정책
- **언제 등장**: 멀티테넌트 보안 (P-5)
- **사용자가 알아둘 것**: 코드에서 회사 ID 빼먹어도 DB 자체가 막아주는 두 번째 안전장치

### TTL (Time To Live)
- **한 줄 풀이**: 데이터가 신선하다고 인정받는 시간 (= 우유 유통기한)
- **언제 등장**: 캐시, InfoHub 응답
- **사용자가 알아둘 것**: 우리는 InfoHub 응답을 6시간 동안만 신뢰함. 그 후엔 ❓ 미확인
